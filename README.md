### About Me

I'm based in Bangalore, currently finishing my B.Tech in Electronics & Communication Engineering, and I spend most of my time working with data — cleaning it, querying it, and turning it into dashboards and reports that make sense to people who aren't going to dig through raw tables themselves.

My core stack is Python and SQL for the analysis side, and Power BI / Tableau for the reporting side. I like the part of the job where a messy spreadsheet or a database full of transactions turns into a dashboard that answers a real business question — revenue trends, customer behavior, product performance, that kind of thing.

Lately I've also been building small AI-powered analytics tools using Google Gemini, mostly around letting non-technical users ask questions of their data in plain English and get a chart or a summary back. It's a good complement to traditional BI work rather than a replacement for it.

I'm currently sharpening my SQL and data modeling skills further, and getting more comfortable with the business side of analytics — not just building a dashboard, but knowing which metrics actually matter.

Looking for a Data Analyst role where I can work close to the business, build things people actually use, and keep learning from real problems instead of tutorials.

#### 📊 E-Commerce Sales Intelligence Hub
`Python` `MySQL` `Power BI` `DAX` `ETL`

**Business problem:** A large e-commerce transaction dataset (390k+ records) needed to be turned into something the business could actually act on — revenue trends, order patterns, customer activity.

**What I did:** Built an end-to-end analytics workflow — cleaned and transformed the raw sales data in Python and MySQL, then built an interactive Power BI dashboard on top of it.

**Impact:** Made it possible to track revenue, orders, customer activity, and product performance in one place, and surfaced buying behavior and sales trends that weren't obvious in the raw data.

`Python` `Streamlit` `Google Gemini AI` `Pandas` `Plotly`

**Business problem:** Non-technical users often have a CSV or Excel file full of useful data but no easy way to explore it without writing code.

**What I did:** Built an AI-powered analytics app that lets users upload a dataset and get data cleaning, interactive dashboards, KPI reporting, and AI-generated summaries and recommendations — including downloadable PDF reports. Deployed it on Streamlit Cloud with secure API key handling.

#### 🗄️ QueryGPT — Universal AI SQL Database Analyst
`Python` `SQLite` `Streamlit` `Google Gemini AI` `SQL` `Plotly`

**Business problem:** Writing SQL is a barrier for a lot of people who still need answers from a database.

**What I did:** Built an app that converts natural language questions into SQL queries against SQLite databases, with automatic schema detection, query execution, interactive visualizations, and AI-generated business insights. Added CSV/PDF export for easy reporting.
